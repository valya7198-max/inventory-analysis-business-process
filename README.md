# Система анализа запасов дистрибьютора автозапчастей

## 📌 Описание проекта
Проект направлен на анализ и оптимизацию бизнес-процессов управления запасами регионального дистрибьютора автозапчастей.

Основная проблема — отсутствие формализованного процесса анализа запасов и высокая доля ручной обработки данных.

---

## 🎯 Цель
Разработка модели бизнес-процессов и требований к информационной системе анализа запасов.

---

## 🔍 AS-IS (как есть)
- Ручной анализ остатков
- Отсутствие регламентов
- Разрозненные системы (Excel, 1С)
- Зависимость от сотрудников

---

## ❗ Выявленные проблемы
- Избыточные запасы
- Дефицит товаров
- Ошибки в принятии решений
- Высокая нагрузка на персонал

---

## 🔄 TO-BE (как должно быть)
- Автоматизированный анализ запасов
- Единая система данных
- Формирование отчетов
- Рекомендации по закупкам

---

## 🧩 Бизнес-процессы
- Анализ товарных запасов
- Принятие решения о пополнении
- Учет товарных позиций
- Формирование отчетности

---

## 📊 Артефакты аналитика
- BPMN диаграммы
- EPC диаграммы
- IDEF0 модели
- Регламенты процессов
- Матрица ответственности

---

## 📋 Требования

### Функциональные:
- Формирование отчетов по остаткам
- Выявление дефицита и излишков
- Поддержка принятия решений по закупкам

### Нефункциональные:
- Высокая точность данных
- Быстрая обработка информации
